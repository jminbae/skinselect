# 스킨셀렉트 (SkinSelect) — 서비스 초안 v0.1

> **피부에 대해 궁금하면, 광고가 아니라 의사의 답을 읽는 곳.**
> 피부과 의사가 실명으로 직접 쓰는 검증된 피부 정보 SNS·매거진 + 원장 퍼스널 브랜딩 플랫폼.

이 저장소는 기획서 v2.3(2026-07-03)과 상세기획서(별책)를 바탕으로 만든 **시각 초안(프로토타입)** 입니다.
비로그인 독자 뷰를 중심으로 IA·디자인 방향을 검증하기 위한 정적 사이트이며, 칼럼·답변 텍스트는 **원장 감수 전의 예시 콘텐츠**입니다.

## 화면 구성 (상세기획서 화면 ID 기준)

| 경로 | 화면 | 비고 |
|---|---|---|
| `/` | HOME-02 비로그인 매거진 큐레이션 뷰 | 오늘의 픽 · 에디터 픽 · 카테고리 · 질문/답변 베스트 · 원장별 최신 |
| `/columns` | EXPL-01 탐색 | 카테고리 8개 필터 |
| `/columns/{slug}` | COL-01 칼럼 상세 | 요약 스니펫 박스 · 저자/감수 라인 · TOC · Q&A 블록 · 출처 · JSON-LD |
| `/columns/topic/{category}` | EXPL-02 주제 허브 | 허브 글 상단 고정 |
| `/doctors` | DOC-00 원장 디렉토리 | 지점·분야 필터, 별점·순위 없음 |
| `/doctors/{slug}` | DOC-01 원장 페이지 | HERO 크로스페이드 · 분야 칩 · 대표 칼럼 · 갤러리/장비(peek 스크롤) · **예약 버튼 없음** |
| `/clinics/{slug}` | CLI-01 지점 페이지 | **'광고' 라벨 상시** · NAP · 진료시간 · 예약 아웃링크 |
| `/community` | COM-01 공개 질문 보드 | 질문↔의사 답변, 칼럼 승화 배지, 고지문 |
| `/about` | 서비스 소개 | 신뢰 원칙 4 + 발행 파이프라인 |

## 설계 원칙 (기획서에서 상속)

- **콘텐츠(정보)와 광고의 URL 분리** — 원장 페이지에 예약 버튼 없음, 병원 실무 정보는 `/clinics/`(광고 라벨)에만
- **비교·랭킹 금지** — 별점·후기·조회수 순위 UI 없음
- **전후사진·치료경험담 미개발** — 갤러리는 인물·공간·장비 한정
- **모든 칼럼에 근거 출처 + 공통 의학 고지문**
- **접근성** — `prefers-reduced-motion` 대응(크로스페이드·카운트업 비활성), WCAG 2.2.2 일시정지 컨트롤

## 기술 스택

- Next.js 15 (App Router, 정적 생성) + TypeScript + Tailwind CSS v4
- 디자인: 미니멀 — 화이트 베이스 + 헤어라인 + 단일 그린 액센트, Pretendard 단일 서체
- 콘텐츠: `src/lib/data/`의 타입드 목업 데이터 (CMS 도입 전 초안 단계)
- 이미지: `scripts/process-images.mjs` — 원본 사진자료 → WebP 최적화

## 실행

```bash
npm install
npm run dev   # http://localhost:3000
npm run build # 정적 빌드 검증
```

이 저장소를 Vercel에 Import하면 추가 설정 없이 배포됩니다 (Framework: Next.js 자동 감지).
